TASK: Fix the UI conditional rendering trap that is causing MetaMask connections to be ignored by the layout state.

CRITICAL BUG FIX:
Currently, the UI is strictly locked behind Privy's 'authenticated' boolean. However, users who connect via external wallets (like MetaMask) populate the 'useWallets().wallets' array without flipping 'authenticated' to true. This causes the UI to display "Connect Wallet" even while Wagmi successfully reads their balances in the background.

EXACT IMPLEMENTATION:

1. CREATE A UNIFIED CONNECTION STATE:
   - In your layout files (Navbar.tsx, Sidebar.tsx), Dashboard (src/app/dashboard/page.tsx), Swap, and Stake pages, you MUST evaluate BOTH Privy states.
   - Import both hooks: 
     `const { authenticated, ready } = usePrivy();`
     `const { wallets } = useWallets();`
   - Create a unified active state boolean:
     `const isUIActive = ready && (authenticated || (wallets && wallets.length > 0));`

2. APPLY CONDITIONAL RENDERING:
   - Replace any strict `{authenticated ? ... : ...}` checks with `{isUIActive ? ... : ...}`.
   - If 'isUIActive' is TRUE: 
     - Sidebar/Navbar: Hide the "Connect Wallet" & "Sign In" buttons. Render the active wallet address and the "Disconnect" button.
     - Dashboard: Hide the "Connect your wallet to see activity" fallback. Render the actual portfolio stats and Recent Activity tables.
   - If 'isUIActive' is FALSE: Show the default disconnected states.

3. EXECUTE:
   - Run 'npm run build' to confirm no type-check errors.
   - Git commit with message: "fix: unify UI render state to accept both Privy authenticated sessions and external wallet connections" and push to main.
